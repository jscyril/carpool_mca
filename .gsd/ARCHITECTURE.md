## Overview

**UniRide** is a college-specific carpooling platform built for Christ University students. It features a **FastAPI** backend with async PostgreSQL + PostGIS and a **Flutter** mobile frontend. The system enforces a tiered identity verification model (phone → email → identity → driver) and uses passwordless OTP authentication. Rides are modeled as a lifecycle state machine with geospatial start/end points.

```
┌──────────────────────────────────────────────┐
│          Flutter App (UniRide)                │
│   Auth → Home → Rides → Driver → Profile     │
├──────────────────────────────────────────────┤
│      REST API (HTTP + Bearer JWT)             │
├──────────────────────────────────────────────┤
│       FastAPI (Async Python 3.x)             │
│   14 Routers  |  Services  |  Schemas        │
├──────────────────────────────────────────────┤
│   PostgreSQL + PostGIS (Async SQLAlchemy)     │
└──────────────────────────────────────────────┘
```

## Data Flow

### Registration Flow
```
Flutter → POST /auth/phone/send-otp → OTPService creates session → SMSService sends OTP
       → POST /auth/phone/verify-otp → OTPService verifies → phone_verified_token (JWT)
       → POST /auth/register (with token + name/gender) → User created → access_token
```

### Login Flow
```
Flutter → POST /auth/login/send-otp → OTPService creates session → SMSService sends OTP
       → POST /auth/login/verify-otp → OTPService verifies → access_token (JWT)
```

### Ride Lifecycle
```
Driver: POST /rides/ (OPEN)
  ↓
Passenger: POST /rides/{id}/request → RideRequest (PENDING)
  ↓
Driver: PUT /rides/{id}/requests/{requestId} (ACCEPTED → RideParticipant created)
  ↓
Status: driver_arriving → driver_arrived → rider_picked_up → ongoing → completed
  ↓
POST /ratings/{rideId} (both sides rate each other)
```

### Verification Ladder
```
Unverified → [phone OTP] → is_phone_verified
           → [college email OTP] → is_email_verified
           → [college ID upload + face photo] → is_identity_verified (via OCR/manual)
           → [licence/RC document] → is_driver_verified (via SurePass or console)
```
